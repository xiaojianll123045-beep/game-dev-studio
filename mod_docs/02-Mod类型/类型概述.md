# Mod 类型概述

游戏模组分为三种核心类型，每种类型有不同的应用场景和开发方式。

## 数据 Mod (Data)

**用途**：修改游戏中的数值、配置和内容定义。

**适用场景**：
- 调整单位属性、资源产出、科技消耗
- 添加新的组件、特质、成就
- 创建自定义事件和合成配方
- 平衡性调整

**特点**：
- 无需编写代码，使用 JSON 格式
- 可被多个模组叠加
- 加载时自动合并到游戏数据

**文件位置**：`mods/你的模组/data/`

## 脚本 Mod (Script)

**用途**：添加自定义逻辑和行为。

**适用场景**：
- 自定义游戏机制和系统
- 响应游戏事件
- 创建新 UI 界面
- 与存档系统交互

**特点**：
- 使用 GDScript 编写
- 通过 `bridge` 对象访问游戏 API
- 支持钩子系统（Hooks）和命令系统
- 可以操作运行时数据

**文件位置**：`mods/你的模组/scripts/`

## 语言 Mod (Language)

**用途**：提供文本翻译和本地化支持。

**适用场景**：
- 翻译游戏界面和提示文本
- 创建自定义语言包
- 为数据模组中的新增内容提供翻译

**特点**：
- 使用 JSON 格式的键值对
- 自动覆盖游戏原文
- 支持所有游戏支持的语种

**文件位置**：`mods/你的模组/locale/`

## 混用模式

一个模组可以同时包含多种类型的内容。例如，一个数据模组可以附带语言包：

```
MyHybridMod/
├── mod.json              # type 为 "data"
├── data/
│   └── balance.json
└── locale/
    └── zh-CN.json
```

## 选择建议

| 你的需求 | 推荐类型 |
|---------|---------|
| 只想改数值和配置 | 数据 Mod |
| 需要添加新功能逻辑 | 脚本 Mod |
| 需要翻译游戏 | 语言 Mod |
| 需要数据 + 逻辑 | 数据 + 脚本混用 |
| 需要逻辑 + 翻译 | 脚本 + 语言混用 |

## 加载顺序

模组的加载顺序由依赖关系决定：

1. 语言 Mod 最先加载，确保文本可用
2. 数据 Mod 其次加载，建立基础配置
3. 脚本 Mod 最后加载，基于数据和文本执行逻辑

依赖通过 `mod.json` 中的 `dependencies` 字段声明。
