# 自定义组件

组件是游戏中构建单位和建筑的基础模块。通过数据模组，你可以创建全新的自定义组件。

## 组件定义

在 `data/components/` 目录下创建组件定义文件：

```json
{
  "components": [
    {
      "id": "reactor_mk3",
      "name": "反应堆 Mk3",
      "description": "高级核聚变反应堆，提供大量能量输出",
      "type": "power",
      "rarity": "epic",
      "tier": 3,
      "stats": {
        "energy_output": 150,
        "heat_generation": 30,
        "health": 200,
        "mass": 50
      },
      "requirements": {
        "tech": "fusion_power",
        "level": 2,
        "resources": {
          "titanium": 100,
          "uranium": 50,
          "credits": 5000
        }
      },
      "effects": [
        {
          "type": "passive",
          "effect": "energy_regen",
          "value": 5
        }
      ],
      "tags": ["advanced", "power", "reactor"]
    }
  ]
}
```

## 组件字段说明

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `id` | String | 是 | 组件唯一标识 |
| `name` | String | 是 | 显示名称 |
| `description` | String | 是 | 描述文本（支持语言包键值） |
| `type` | String | 是 | 组件类型（`power`/`weapon`/`armor`/`utility` 等） |
| `rarity` | String | 否 | 稀有度（`common`/`rare`/`epic`/`legendary`） |
| `tier` | Integer | 否 | 科技等级 |
| `stats` | Object | 是 | 属性数值 |
| `requirements` | Object | 否 | 解锁条件 |
| `effects` | Array | 否 | 特殊效果列表 |
| `tags` | Array | 否 | 标签 |

## 组件类型

| 类型 | 用途 |
|------|------|
| `power` | 能量源，提供能源输出 |
| `weapon` | 武器，提供攻击能力 |
| `armor` | 装甲，提供防御能力 |
| `shield` | 护盾，提供额外防护 |
| `engine` | 引擎，提供移动能力 |
| `sensor` | 传感器，提供探测范围 |
| `utility` | 功能性组件 |
| `special` | 特殊效果组件 |

## 效果系统

组件可以附带多种效果：

```json
{
  "effects": [
    {
      "type": "passive",
      "effect": "armor_penetration",
      "value": 0.2,
      "condition": "health_below_50"
    },
    {
      "type": "active",
      "effect": "emergency_shield",
      "value": 500,
      "cooldown": 60.0,
      "duration": 10.0
    },
    {
      "type": "aura",
      "effect": "repair_aura",
      "value": 5,
      "radius": 100.0
    }
  ]
}
```

## 通过脚本访问组件

```gdscript
func _ready():
    # 注册自定义组件逻辑
    bridge.register_component_behavior("reactor_mk3", self, "_on_reactor_tick")

func _on_reactor_tick(unit_id, component_data):
    if component_data.energy_output > 100:
        bridge.log_info("反应堆过载！")
        bridge.modify_unit_stat(unit_id, "heat", component_data.heat_generation * 1.5)
```

## 注意事项

- 组件 ID 必须唯一，建议添加模组前缀如 `mymod_component_name`
- 数值范围参考游戏内同类组件的数值
- `effects` 需要在脚本中实现对应的行为逻辑才有效
