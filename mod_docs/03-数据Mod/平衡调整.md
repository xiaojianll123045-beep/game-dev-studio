# 平衡调整

数据模组最常见的用途是调整游戏平衡性。通过 JSON 配置文件，你可以修改几乎所有的游戏数值。

## 文件位置

将平衡调整文件放在 `data/` 目录下，文件名任意，但建议使用有意义的名称：

```
data/
├── balance.json
├── economy.json
└── combat.json
```

## 基础格式

```json
{
  "balance": {
    "economy": {
      "base_income": 10,
      "income_per_pop": 0.5,
      "starting_funds": 200,
      "interest_rate": 0.05
    },
    "combat": {
      "damage_multiplier": 1.0,
      "defense_reduction": 0.1,
      "crit_chance": 0.15,
      "crit_multiplier": 2.0
    }
  }
}
```

## 常见调整项

### 经济调整

```json
{
  "balance": {
    "economy": {
      "base_income": 15,
      "income_per_pop": 0.8,
      "starting_funds": 500,
      "interest_rate": 0.03,
      "trade_income_modifier": 1.2,
      "maintenance_cost_modifier": 0.9
    }
  }
}
```

### 单位属性调整

```json
{
  "balance": {
      "units": {
          "scout": {
              "health": 30,
              "attack": 5,
              "defense": 2,
              "speed": 4.0,
              "cost": 50,
              "upkeep": 1
          },
          "warrior": {
              "health": 100,
              "attack": 12,
              "defense": 8,
              "speed": 2.5,
              "cost": 150,
              "upkeep": 3
          }
      }
  }
}
```

### 科技消耗调整

```json
{
  "balance": {
    "research": {
      "base_cost_multiplier": 1.0,
      "cost_per_tech": 50,
      "discount_per_era": 0.1
    }
  }
}
```

## 乘法与加法修饰

部分字段支持修饰符模式：

```json
{
  "balance": {
    "economy": {
      "income_modifiers": {
        "per_population": {"type": "add", "value": 0.5},
        "per_city": {"type": "multiply", "value": 1.1},
        "difficulty_bonus": {"type": "add", "value": {"easy": 5, "normal": 0, "hard": -5}}
      }
    }
  }
}
```

## 数据合并规则

当多个模组修改同一数值时，遵循以下规则：

1. 基础类型（数字、字符串）直接覆盖
2. 对象类型（Object）进行深度合并
3. 数组类型（Array）根据配置合并或替换
4. 加载顺序靠后的模组优先级更高

## 验证与测试

使用 `bridge` 在运行时检视数值：

```gdscript
func _ready():
    var income = bridge.get_game_value("economy.base_income")
    bridge.log_info("当前基础收入: " + str(income))
```

## 注意事项

- 修改前请记录原始数值，方便恢复
- 注意数值溢出（尤其是大数值乘法）
- 平衡性修改建议分步进行，多次测试
