# 黑天鹅事件

黑天鹅事件是游戏中罕见但影响巨大的特殊事件。

## 事件特点

- **低概率**：发生概率极低（通常 < 1%）
- **高影响**：对游戏进程产生重大影响
- **不可预测**：没有固定的触发模式
- **记忆深刻**：玩家会长期记住此次经历

## 触发条件

```json
{
  "trigger_conditions": {
    "min_turn": 100,
    "max_turn": 500,
    "required_techs": ["tech_a", "tech_b"],
    "required_traits": ["trait_x"],
    "required_resources": {
      "credits": 10000,
      "influence": 500
    },
    "exclude_conditions": [
      {"type": "has_event_occurred", "event_id": "previous_black_swan"}
    ],
    "global_flags": {
      "era": "information_age",
      "world_crisis": false
    }
  }
}
```

## 影响范围

黑天鹅事件的影响可以是多方面的：

| 影响类型 | 说明 |
|---------|------|
| `immediate` | 立即生效的影响 |
| `permanent` | 永久性改变游戏规则 |
| `temporary` | 持续若干回合的效果 |
| `choice_dependent` | 根据玩家选择产生的不同结果 |

## 脚本集成

```gdscript
func _ready():
    bridge.register_hook("on_black_swan", self, "_on_black_swan")

func _on_black_swan(event_id, data):
    if event_id == "technological_singularity":
        bridge.log_info("黑天鹅事件触发：技术奇点！")
        bridge.set_global_flag("singularity_achieved", true)
        bridge.unlock_all_techs(data.player_id)

    # 记录黑天鹅事件到统计数据
    bridge.increment_stat("black_swan_count", 1)
```

## 注意事项

- 黑天鹅事件的触发概率应该非常低，以保持其稀有性
- 每个游戏流程中，黑天鹅事件最多触发 1-2 次
- 要确保事件效果与游戏平衡性兼容
- 建议为黑天鹅事件编写特殊的叙事文本和视觉效果
- 可通过 `exclude_conditions` 防止同一局中重复触发相同事件
