# 自定义事件

创建游戏中触发的危机事件和随机事件。

## 事件定义

在 `data/events/` 目录下创建事件文件：

```json
{
  "events": [
    {
      "id": "asteroid_strike",
      "type": "crisis",
      "name": "小行星撞击",
      "description": "一颗巨型小行星正朝星球袭来！",
      "trigger": {
        "condition": "random",
        "chance": 0.05,
        "min_turn": 50,
        "repeatable": false,
        "conditions": [
          {"type": "has_colony", "value": true},
          {"type": "tech_level", "min": 2, "max": 5}
        ]
      },
      "stages": [
        {
          "id": "approaching",
          "duration": 10,
          "effects": {
            "global_alert": true,
            "production_penalty": 0.3
          },
          "choices": [
            {
              "id": "defend",
              "text": "发射导弹拦截",
              "requirements": {"tech": "missile_tech", "level": 2},
              "success_chance": 0.7,
              "success": {
                "next_stage": "destroyed"
              },
              "failure": {
                "next_stage": "impact"
              }
            },
            {
              "id": "evacuate",
              "text": "紧急疏散",
              "success_chance": 0.9,
              "success": {
                "next_stage": "evacuated"
              },
              "failure": {
                "next_stage": "impact"
              }
            }
          ]
        },
        {
          "id": "impact",
          "duration": 1,
          "effects": {
            "colony_damage": 0.8,
            "population_loss": 0.5
          },
          "next": "aftermath"
        }
      ]
    }
  ]
}
```

## 事件字段

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `id` | String | 是 | 事件唯一标识 |
| `type` | String | 是 | 事件类型（`crisis`/`random`/`story`/`chain`） |
| `name` | String | 是 | 事件名称 |
| `description` | String | 是 | 事件描述 |
| `trigger` | Object | 是 | 触发条件配置 |
| `stages` | Array | 是 | 事件阶段列表 |

## 事件类型

| 类型 | 说明 |
|------|------|
| `crisis` | 危机事件，通常有负面影响 |
| `random` | 随机事件，可好可坏 |
| `story` | 剧情事件，按流程触发 |
| `chain` | 连锁事件，包含多个关联事件 |

## 触发条件

```json
{
  "trigger": {
    "condition": "and",
    "conditions": [
      {"type": "turn_count", "min": 30, "max": 100},
      {"type": "has_building", "id": "space_port"},
      {"type": "resource_level", "resource": "credits", "min": 1000}
    ]
  }
}
```

## 脚本集成

```gdscript
func _ready():
    bridge.register_hook("on_event_triggered", self, "_on_event_triggered")
    bridge.register_hook("on_event_choice", self, "_on_event_choice")

func _on_event_triggered(event_id, stage_id):
    if event_id == "asteroid_strike":
        bridge.log_info("小行星事件触发！")
        bridge.set_global_flag("asteroid_incoming", true)

func _on_event_choice(event_id, choice_id, result):
    if event_id == "asteroid_strike" and choice_id == "defend":
        if result == "success":
            bridge.log_info("拦截成功！")
            bridge.add_resource("player", "experience", 100)
        else:
            bridge.log_info("拦截失败...")
            bridge.start_event("asteroid_strike", "impact")
```

## 注意事项

- 事件 ID 需要唯一，建议加模组前缀
- `trigger` 中的 `chance` 是每次检查时的触发概率
- 连锁事件可以通过 `next` 字段连接
- 复杂事件逻辑建议结合脚本模组实现
