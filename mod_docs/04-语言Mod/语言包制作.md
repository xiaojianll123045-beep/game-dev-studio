# 语言包制作

语言模组用于翻译游戏文本或添加新的语言支持。

## 文件结构

语言包文件放在 `locale/` 目录下，以语言代码命名：

```
locale/
├── zh-CN.json       # 简体中文
├── zh-TW.json       # 繁体中文
├── en.json          # 英文
├── ja.json          # 日文
└── ko.json          # 韩文
```

## mod.json 配置

```json
{
  "version": "1.0.0",
  "author": "Translator",
  "type": "language"
}
```

无需填写 `name` 和 `description`，语言模组名称会从游戏语言系统自动获取。

## JSON 格式

语言包使用扁平的键值对格式（无嵌套 `"strings"` 或 `"locale"` 包装）：

```json
{
  "ui.main_menu.title": "游戏工作室",
  "ui.main_menu.new_game": "新游戏",
  "ui.main_menu.load_game": "加载游戏",
  "ui.main_menu.settings": "设置",
  "ui.main_menu.quit": "退出",
  "ui.main_menu.mods": "模组",
  "ui.settings.audio": "音频设置",
  "ui.settings.video": "视频设置",
  "ui.settings.controls": "控制设置",
  "ui.settings.language": "语言",
  "ui.load_title": "加载中...",
  "menu.about": "关于",
  "set.display": "显示设置"
}
```

游戏读取逻辑（`ModManager.ApplyLanguageMod`）：
1. 打开 `locale/` 目录
2. 遍历每个 `.json` 文件，去掉后缀作为语言代码
3. 解析 JSON 并调用 `Loc.MergeDictionary(langCode, flatDict)` 合并到游戏本地化系统

**不要**使用以下格式（游戏不会正确解析）：

```json
{
  "locale": "zh-CN",
  "strings": {
    "key": "value"
  }
}
```

## 键值命名规范

推荐使用点分命名法：

```
<域>.<子域>.<具体项>.<属性>
```

| 域 | 用途 |
|----|------|
| `ui` | 界面文本 |
| `menu` | 菜单文本 |
| `set` | 设置文本 |
| `dialog` | 对话框文本 |
| `tooltip` | 工具提示 |
| `notification` | 通知消息 |
| `misc` | 杂项文本 |

建议参考游戏中已有的键名模式来命名。

## 变量替换

文本中可以使用 `{varName}` 进行动态替换，由 `Loc.TrF()` 处理：

```json
{
  "notification.attack": "{attacker} 正在攻击 {defender}！",
  "notification.income": "获得 {amount} 资金"
}
```

## 脚本中读取本地化文本

GDScript 模组通过 `Loc` 自动加载对象读取本地化文本，这不属于 ModBridge 的范围：

```gdscript
var text = Loc.Tr("ui.main_menu.title")
var text_with_vars = Loc.TrF("notification.attack", {"attacker": "玩家", "defender": "敌人"})
```

语言模组仅负责提供 JSON 数据，无需编写 GDScript。

## 注意事项

- 语言包文件编码必须为 UTF-8（无 BOM）
- 确保所有键值在游戏原文中都有对应的引用
- 多个语言模组合并时，后加载的覆盖先加载的
- 建议为数据模组中新增的内容提供翻译，否则会显示键值原文
- 测试时切换到对应语言以验证翻译是否正确显示
- 请参考 `mod_examples/` 中的语言模组示例
