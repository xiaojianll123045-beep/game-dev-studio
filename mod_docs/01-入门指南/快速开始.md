# 快速开始 — 模组制作入门

欢迎来到游戏模组开发指南！本文档将帮助你快速上手创建第一个模组。

## 前置条件

- 游戏本体已安装
- 文本编辑器（推荐 VS Code、Sublime Text 或 Notepad++）
- 基本的 JSON 和 GDScript 知识

## 模组的基本结构

一个标准的模组包含以下文件结构：

```
MyMod/
├── mod.json          # 模组配置文件（必填）
├── data/             # 数据模组文件
│   ├── balance.json
│   └── components/
├── scripts/          # 脚本模组文件
│   └── main.gd
└── locale/           # 语言包文件
    └── zh-CN.json
```

## 创建你的第一个模组

### 第 1 步：创建模组文件夹

在游戏目录的 `mods/` 文件夹下创建一个新文件夹，命名为 `MyFirstMod/`。

### 第 2 步：编写 mod.json

```json
{
  "id": "my_first_mod",
  "name": "我的第一个模组",
  "version": "1.0.0",
  "author": "你的名字",
  "description": "这是我的第一个模组！",
  "type": "data",
  "entry_point": "scripts/main.gd",
  "dependencies": [],
  "compatibility": {
    "game_version": ">=1.0.0"
  }
}
```

### 第 3 步：加载测试

将模组文件夹放入 `mods/` 目录后，启动游戏。如果模组配置正确，游戏主菜单的"模组"选项中会显示你的模组。

## 与游戏引擎交互

通过 `bridge` 全局对象与游戏进行交互。所有 API 调用都通过 `bridge.xxx()` 形式完成。

### 在脚本中获取桥接对象

```gdscript
# 通过全局自动加载（推荐）
bridge.log_info("模组已加载")

# 或通过节点路径获取
var mod_bridge = get_node("/root/GameManager/ModBridge")
mod_bridge.log_info("模组已加载")
```

### 模组生命周期

```gdscript
extends Node

# 模组初始化
func _ready():
    bridge.log_info("模组初始化完成")
    bridge.register_hook("on_game_start", self, "_on_game_start")

# 游戏启动时调用
func _on_game_start(data):
    bridge.log_info("游戏已启动")
    bridge.register_command("my_command", self, "_on_my_command")

func _on_my_command(args):
    bridge.log_info("执行自定义命令: " + str(args))
```
