# 快速开始 — 模组制作入门

欢迎来到游戏模组开发指南！本文档将帮助你快速上手创建第一个模组。

## 前置条件

- 游戏本体已安装
- 文本编辑器（推荐 VS Code、Sublime Text 或 Notepad++）
- 基本的 JSON 和 GDScript 知识

## 模组的基本结构

```
MyMod/
├── mod.json              # 模组配置文件（必填）
├── mod_zh-CN.json        # 中文名称/描述（可选）
├── mod_en.json           # 英文名称/描述（可选）
├── scripts/              # 脚本模组文件（可选）
│   └── main.gd
├── data/                 # 数据模组文件（可选）
│   └── balance.json
└── locale/               # 语言包文件（可选）
    └── zh-CN.json
```

## 创建你的第一个模组

### 第 1 步：创建模组文件夹

在游戏目录的 `mods/` 文件夹下创建一个新文件夹，模组的 **id 就是文件夹名称**。例如 `mods/MyFirstMod/`，则 id 为 `MyFirstMod`。

### 第 2 步：编写 mod.json

```json
{
  "version": "1.0.0",
  "author": "你的名字",
  "type": "script"
}
```

`version`、`author`、`type` 为必填字段。`name` 和 `description` 写在独立的 `mod_{lang}.json` 文件中（例如 `mod_zh-CN.json`）：

```json
{
  "name": "我的第一个模组",
  "description": "这是我的第一个模组！"
}
```

**注意**：`mod.json` 中没有 `entry_point` 字段（代码会忽略它）。

### 第 3 步：编写脚本

所有放在 `scripts/` 目录下的 `.gd` 文件都会被加载。每个脚本都需实现 `OnLoad(gm, bridge)` 函数作为入口：

```gdscript
extends Node

func OnLoad(gm, bridge):
    var b = Engine.get_singleton("ModBridge")
    b.log("模组已加载！")
    b.add_money(50000)
```

**注意**：
- 入口函数是 `OnLoad(gm, bridge)`，**不是** `_ready()`
- `bridge` 参数已弃用（始终为 `null`），请通过 `Engine.get_singleton("ModBridge")` 获取桥接对象
- 详细 API 列表见 [ModBridge_API_参考.md](../03-API参考/ModBridge_API_参考.md)

### 第 4 步：加载测试

将模组文件夹放入 `mods/` 目录后，启动游戏。如果模组配置正确，游戏会在加载时自动调用 `OnLoad`。

按 **F12** 或 **`~`** 打开控制台，输入 `mod_log` 查看模组日志，输入 `help` 查看命令列表。

## 数据模组示例

`data/balance.json` 使用扁平键值对：

```json
{
  "project.init.gameplay_base": 40,
  "project.init.dev_speed": 1.0,
  "economy.start_funds": 100000
}
```

不要使用嵌套 JSON 对象。参考 `mod_examples/` 中的实际示例。
