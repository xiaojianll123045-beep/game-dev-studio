# 脚本系统

深入了解游戏的脚本系统架构。

## 系统架构

脚本系统采用分层架构：

```
游戏引擎
  └─ GameManager (自动加载)
      └─ ModBridge (桥接层)
          └─ ModLoader (模组加载器)
              ├─ DataLoader (数据加载)
              ├─ ScriptRunner (脚本执行)
              └─ LocaleLoader (语言加载)
```

## 模组加载流程

```
1. 游戏启动
2. ModLoader 扫描 mods/ 目录
3. 解析所有 mod.json
4. 解析依赖关系，排序
5. 语言模组加载
6. 数据模组加载与合并
7. 脚本模组实例化
8. 调用 _ready()
9. 触发 on_game_start 钩子
```

## 沙箱环境

脚本模组运行在受控的沙箱环境中：

```gdscript
# 允许的操作
bridge.log_info("允许")
bridge.get_game_value("path")  # 允许只读访问

# 被限制的操作
# OS.execute(...)        # 不允许
# Directory.remove(...)  # 不允许
# File.open(path)        # 只允许 mods/ 目录
```

## 桥接层 (ModBridge)

`ModBridge` 是 `GameManager` 的子节点，作为游戏和模组之间的中介：

```gdscript
# 通过全局自动加载（推荐）
bridge.log_info("Hello")

# 通过节点路径获取桥接对象
var mod_bridge = get_node("/root/GameManager/ModBridge")
if mod_bridge:
    mod_bridge.log_info("Hello from path")
```

### 桥接层职责

- API 转发：将模组调用转发到游戏引擎
- 权限检查：验证模组是否有权限执行操作
- 数据序列化：处理模组数据的存储与读取
- 事件路由：分发钩子事件到注册的模组

## 生命周期管理

```gdscript
extends Node

# 加载阶段
func _init():
    bridge.log_debug("模组正在初始化")

func _ready():
    bridge.log_info("模组就绪")
    bridge.register_hook("on_game_start", self, "_on_game_start")

# 运行阶段
func _on_game_start(data):
    bridge.log_info("游戏开始")

# 卸载阶段
func _exit_tree():
    bridge.log_info("模组正在卸载")
    bridge.unregister_hook("on_game_start", self)
```

## 错误隔离

单个模组的错误不会影响其他模组或游戏本身：

```gdscript
func _ready():
    # 这个错误不会导致游戏崩溃
    bridge.safe_call("non_existent_function")

    # 钩子中的错误会被捕获并记录
    bridge.register_hook("on_tick", self, "_risky_function")

func _risky_function(delta):
    # 即使这里崩溃了，也只是这个钩子被禁用
    var x = 1 / 0  # 除零错误
```

## 模组间通信

模组之间可以通过钩子和命令系统通信：

```gdscript
# 模组 A — 提供服务
func _ready():
    bridge.register_command("mod_a_convert_currency", self, "_convert")

func _convert(args):
    var amount = args.amount * 2.5
    return {"result": amount}

# 模组 B — 调用服务
func _ready():
    bridge.register_hook("on_game_start", self, "_init_service")

func _init_service(data):
    var result = bridge.execute_command("mod_a_convert_currency", {"amount": 100})
    if result:
        bridge.log_info("转换结果: " + str(result.result))
```

## 性能监控

```gdscript
func _ready():
    # 启用性能监控
    bridge.register_hook("on_tick", self, "_monitor_performance")

var _last_time := 0.0
var _frame_count := 0

func _monitor_performance(delta):
    _frame_count += 1
    _last_time += delta

    if _last_time >= 1.0:
        bridge.log_debug("FPS: " + str(_frame_count))
        _frame_count = 0
        _last_time = 0.0
```

## 注意事项

- 游戏使用 Godot 4，GDScript 语法以 Godot 4 为准
- `bridge` 是自动加载的全局对象，在 `_ready()` 中即可访问
- 模组脚本不能访问 `res://` 中未授权的路径
- 建议将模组工具函数放在独立的脚本文件中并 `preload`
- 使用 `bridge.safe_call` 调用可能不存在的游戏内部函数
