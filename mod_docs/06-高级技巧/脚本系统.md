# 脚本系统

深入了解游戏的 GDScript 模组架构。

## 系统架构

```
GameManager (自动加载)
  └─ ModManager (静态单例)
      ├─ ModManifest.Load() → 解析 mod.json
      ├─ ApplyDataMod()     → 加载 data/ JSON 数据
      ├─ ApplyScriptMod()   → 加载 scripts/ GDScript 文件
      └─ ApplyLanguageMod()  → 加载语言包
```

- `ModManager` 是静态类，通过 `Engine.GetSingleton("ModManager")` 访问
- 扫描 `mods/` 目录，解析所有 `mod.json`，按依赖拓扑排序
- 风险扫描通过 `ScanAndConfirmMods` 实现，中风险及以上模组会弹出警告确认

## 脚本模组加载流程

```
1. 游戏启动
2. ModManager 扫描 mods/ 目录
3. 解析所有 mod.json，按依赖拓扑排序
4. 数据模组加载 (ApplyDataMod)
5. 脚本模组加载 (ApplyScriptMod)
6. 语言模组加载 (ApplyLanguageMod)
```

## ApplyScriptMod 细节

- 加载模组 `scripts/` 目录下的 **所有 `.gd` 文件**
- 每个 `.gd` 文件创建一个 `Node` 子节点，挂载脚本
- 加载完成后调用每个脚本的 `OnLoad(gm, bridge)` 方法
- 已激活的脚本模组记录在 `ActiveScriptMods` 中

## 沙箱环境

GDScript 运行在 Godot 的标准脚本沙箱中，没有额外的限制层。模组只能通过 `bridge` 对象与游戏交互，无权访问游戏内部的 C# 对象或方法（除非游戏主动暴露）。

```
# 允许的操作
bridge.log("Hello")           # 日志
bridge.get_setting(...)        # 读取设置
bridge.register_endpoint(...)  # 注册通信端点

# 不被推荐的操作
# OS.execute(...)              # 不建议，但技术上不受限
# 直接访问 /root/GameManager   # 可能绕过 bridge，但不保证稳定性
```

## Bridge 单例

`bridge` 是 `Engine` 的自动加载单例，通过 `Engine.get_singleton("ModBridge")` 获取。

```gdscript
# 推荐方式（bridge 自动可用）
bridge.log("Hello from mod")

# 显式获取
var bridge = Engine.get_singleton("ModBridge")
bridge.log("Hello via singleton")
```

## 生命周期

```gdscript
extends Node

# mod.json 中 scripts 目录下的每个 .gd 文件都作为一个独立的脚本节点加载
# 加载完成后调用 OnLoad

func OnLoad(gm, bridge):
    bridge.log("模组已加载")
    # 此处可以执行初始化逻辑
    # 注意：没有钩子注册，没有命令注册
    # 需要模组间通信时使用 register_endpoint

# 模组被禁用时调用
func OnUnload():
    print("模组正在卸载")
    # 清理工作在此完成
```

## 错误隔离

每个模组的脚本运行在独立的 `Node` 节点下，某个模组的运行时错误不会影响其他模组或游戏本身：

```gdscript
func OnLoad(gm, bridge):
    # 如果这里抛出错误，仅当前脚本节点受影响
    # 其他模组和游戏正常运行
    bridge.log("加载中...")
    # var x = 1 / 0  # 仅当前模组崩溃
```

## 模组间通信 (ModCommAPI)

GDScript 模组通过 ModCommAPI 进行通信，而非钩子系统：

```gdscript
# 模组 A — 注册端点（接收消息）
bridge.register_endpoint("mod_a.convert", func(data):
    return {"result": data.amount * 2.5}
)

# 模组 B — 发送消息给特定模组
func OnLoad(gm, bridge):
    var result = bridge.send_message("mod_a", "mod_a.convert", {"amount": 100})
    if result != null:
        bridge.log("转换结果: " + str(result.result))

# 广播消息给所有模组
bridge.broadcast_message("system.announce", {"msg": "大家好！"})
```

| 方法 | 说明 |
|------|------|
| `register_endpoint(path, callable)` | 注册一个消息端点，`path` 建议格式 `mod_id.endpoint_name` |
| `send_message(target_mod, endpoint, data)` | 向指定模组的端点发送消息，返回端点的返回值 |
| `broadcast_message(endpoint, data)` | 向所有模组的端点广播消息 |

## 数据持久化

```gdscript
# 简单键值对存储（独立于游戏存档）
bridge.set_setting("my_mod", "volume", 0.8)
var vol = bridge.get_setting("my_mod", "volume", 0.5)
```

| 方法 | 说明 |
|------|------|
| `get_setting(mod_id, key, fallback)` | 读取设置值，不存在时返回 fallback |
| `set_setting(mod_id, key, value)` | 写入设置值 |

详见 `存档处理.md`。

## 控制台命令

- 按 `F12` 或 `~` 键打开 ModConsole
- 控制台命令由 C# 模组注册，GDScript 模组不能直接注册命令
- GDScript 模组可以通过 `ModConsole` 的输出功能调试日志

## 注意事项

- 游戏使用 Godot 4，GDScript 语法以 Godot 4 为准
- 入口方法是 `OnLoad(gm, bridge)`，**不是** `_ready()`
- 每个 `.gd` 文件独立加载，可以互相 `preload` 或通过 ModCommAPI 通信
- 模组脚本不能访问 `res://` 中未授权的路径
- 建议将模组工具函数放在独立脚本中并 `preload`
- 数据模组的 JSON 文件放在 `data/` 目录下，会在脚本加载前合并到游戏数据中
