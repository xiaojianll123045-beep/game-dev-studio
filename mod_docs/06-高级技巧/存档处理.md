# 存档处理

GDScript 模组的数据持久化指南。

## 两种持久化机制

### 1. 简单设置 (Settings)

通过 `bridge.get_setting()` / `bridge.set_setting()` 存储键值对数据，适合小型配置。数据保存在 `user://mods/{modId}/settings.json`，**不与游戏存档绑定**（删除游戏存档不会清除此数据）。

```gdscript
# 写入设置
bridge.set_setting("my_mod", "player_name", "Alice")
bridge.set_setting("my_mod", "high_score", 9999)
bridge.set_setting("my_mod", "unlocked_features", ["feature_a", "feature_b"])

# 读取设置
var name = bridge.get_setting("my_mod", "player_name", "Unknown")
var score = bridge.get_setting("my_mod", "high_score", 0)
var features = bridge.get_setting("my_mod", "unlocked_features", [])

bridge.log("玩家: " + str(name) + " 得分: " + str(score))
```

| 方法 | 签名 | 说明 |
|------|------|------|
| `get_setting` | `get_setting(mod_id, key, fallback)` | 读取指定模组的设置值，不存在时返回 fallback。`mod_id` 与 `mod.json` 中的 `id` 字段一致 |
| `set_setting` | `set_setting(mod_id, key, value)` | 写入指定模组的设置值。支持 String、int、float、bool、Array、Dictionary |

### 2. 游戏存档数据 (Game Save Data) — C# 仅限

C# 原生模组可以通过 `ModAPI.RegisterSaveHandler` 注册序列化/反序列化回调，将自定义数据嵌入游戏存档文件：

```csharp
ModAPI.RegisterSaveHandler("my_csharp_mod",
    () => {
        // 序列化：返回 Dictionary<string, object> 表示要保存的数据
        return new Dictionary<string, object> {
            { "version", "1.0.0" },
            { "flags", _gameFlags }
        };
    },
    (Dictionary<string, object> data) => {
        // 反序列化：从 data 中恢复状态
        _gameFlags = data.ContainsKey("flags") ? (Dictionary)data["flags"] : new Dictionary();
    }
);
```

**GDScript 模组无法使用此机制。** 数据会嵌入 `.sav` 文件中，删除存档时一并清除。

## 数据模组方案

使用 `data/` 目录下的 JSON 文件提供静态配置数据，无需持久化机制：

```json
// mods/my_mod/data/items.json
{
    "items": [
        { "id": "custom_tool", "name": "自定义工具", "cost": 500 }
    ]
}
```

JSON 文件通过 `ApplyDataMod` 在游戏启动时合并到主数据表中，适用于不需要运行时保存的配置数据。

## 完整示例

```gdscript
extends Node

const MOD_ID = "my_mod"

func OnLoad(gm, bridge):
    bridge.log("模组加载完成")

    # 读取已有设置
    var start_count = bridge.get_setting(MOD_ID, "load_count", 0)
    start_count += 1
    bridge.set_setting(MOD_ID, "load_count", start_count)

    bridge.log("这是第 " + str(start_count) + " 次加载")

    # 注册通信端点，供其他模组查询设置值
    bridge.register_endpoint(MOD_ID + ".get_setting", func(data):
        return bridge.get_setting(MOD_ID, data.key, data.fallback)
    )
```

## 注意事项

- `get_setting` / `set_setting` 是 GDScript 模组**唯一**的持久化方式
- `bridge.save_mod_data()`、`bridge.load_mod_data()`、`bridge.delete_mod_data()` **不存在**，不要使用
- C# 模组的 `RegisterSaveHandler` 数据会随游戏存档保存/删除；`set_setting` 数据独立于存档
- Settings 数据建议控制在合理大小（< 1MB），不要存储大型资源
- 不要在每帧调用 `set_setting`（频繁写入磁盘影响性能）
- 静态配置数据放在 `data/` JSON 文件中，而非通过设置存储
- 设置文件使用 JSON 格式纯文本存储，不要存放敏感信息
