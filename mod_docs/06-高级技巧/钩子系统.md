# 钩子系统 (C# ModAPI)

钩子系统是 C# 原生模组响应游戏事件的核心机制。**GDScript 模组无法直接注册钩子**，只能通过 ModCommAPI（`register_endpoint`/`send_message`）进行模组间通信。

## 架构概览

- 钩子由 `ModAPI.GameHook` 枚举定义，共 68 个预定义钩子
- C# 模组通过静态方法注册回调，游戏在特定时机触发
- 支持普通回调（Action）和带上下文回调（Action\<HookContext\>）
- 支持取消钩子（Func\<bool\> / Func\<HookContext, bool\>），用于阻止游戏行为

## 注册钩子

### Action 钩子（无上下文）

```csharp
ModAPI.RegisterActionHook(ModAPI.GameHook.AfterMonthEnd, OnMonthEnd);

private static void OnMonthEnd()
{
    ModAPI.LogInfo("月份结束");
}
```

### Action 钩子（带上下文）

```csharp
ModAPI.RegisterActionHookEx(ModAPI.GameHook.BeforeMonthEnd, OnBeforeMonthEnd);

private static void OnBeforeMonthEnd(HookContext ctx)
{
    int month = ctx.Get<int>("month", 0);
    ctx.Set("modified", true);
    ModAPI.LogInfo($"处理月份 {month}");
}
```

### 取消钩子

```csharp
ModAPI.RegisterCancelHook(ModAPI.GameHook.OnGameSave, CanSave);

private static bool CanSave()
{
    return !_isBusy;
}
```

### 取消钩子（带上下文）

```csharp
ModAPI.RegisterCancelHookEx(ModAPI.GameHook.OnEmployeeLeave, CanEmployeeLeave);

private static bool CanEmployeeLeave(HookContext ctx)
{
    var empId = ctx.Get<int>("employeeId", 0);
    return !_criticalWorkers.Contains(empId);
}
```

## 触发钩子

```csharp
// 直接触发
ModAPI.FireHooks(ModAPI.GameHook.AfterMonthEnd);

// 带数据触发
var data = new Dictionary<string, object> { { "month", 3 }, { "year", 2024 } };
ModAPI.FireHooks(ModAPI.GameHook.BeforeMonthEnd, data);
```

## 取消检测

在执行可能被取消的操作前，先检查是否已被取消：

```csharp
if (!ModAPI.IsCancelled(ModAPI.GameHook.OnEmployeeLeave))
{
    // 执行解雇逻辑
    FireEmployee(empId);
}
```

## HookContext

`HookContext` 是传递数据的容器：

| 方法 | 说明 |
|------|------|
| `T Get<T>(string key, T fallback)` | 读取上下文数据，key 不存在时返回 fallback |
| `void Set(string key, object val)` | 写入上下文数据 |

## GDScript 模组的替代方案

GDScript 模组无法直接注册 GameHook，但可以通过 ModCommAPI 实现模组间事件驱动：

```gdscript
# 注册一个端点（相当于"监听"）
bridge.register_endpoint("my_mod.on_update", func(data):
    bridge.log("收到更新通知: " + str(data))
)

# 向特定模组发送消息（相当于"触发事件"）
bridge.send_message("other_mod", "other_mod.do_something", {"value": 42})

# 广播消息给所有模组
bridge.broadcast_message("system.notification", {"msg": "更新完成"})
```

## 注意事项

- 钩子回调应尽量轻量，避免长时间阻塞
- Action 钩子不要修改游戏状态（只读观察）
- 取消钩子返回 `true` 表示阻止操作，返回 `false` 表示允许
- 带 Ex 后缀的注册方法 （`RegisterActionHookEx`/`RegisterCancelHookEx`） 可通过 HookContext 传递数据
- 钩子执行顺序按模组加载顺序，依赖模组优先
- 多个取消钩子中任意一个返回 `true` 则操作被阻止
