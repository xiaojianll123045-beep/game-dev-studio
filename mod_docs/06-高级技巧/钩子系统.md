# 钩子系统

钩子（Hook）是脚本模组响应游戏事件的核心机制。

## 工作原理

钩子系统基于观察者模式工作：

1. 模组注册钩子：声明想监听的事件
2. 游戏触发事件：在特定时机广播事件
3. 模组响应事件：注册的回调方法被自动调用

## 注册钩子

```gdscript
func _ready():
    bridge.register_hook("on_unit_destroyed", self, "_on_unit_destroyed")
```

参数说明：

| 参数 | 类型 | 说明 |
|------|------|------|
| `hook_name` | String | 钩子名称 |
| `target_obj` | Object | 回调方法所属的对象 |
| `method_name` | String | 回调方法名称 |

## 取消注册

```gdscript
bridge.unregister_hook("on_unit_destroyed", self)
```

在模组卸载时自动清理已注册的钩子，但建议显式取消注册。

## 内置钩子

### 游戏生命周期

| 钩子 | 数据 | 说明 |
|------|------|------|
| `on_game_start` | `{player_name}` | 新游戏开始时 |
| `on_game_load` | `{slot}` | 加载存档时 |
| `on_game_end` | `{result}` | 游戏结束时 |
| `on_tick` | `{delta}` | 每帧更新 |
| `on_turn_start` | `{turn}` | 回合开始时 |
| `on_turn_end` | `{turn}` | 回合结束时 |

### 单位相关

| 钩子 | 数据 | 说明 |
|------|------|------|
| `on_unit_created` | `{unit_id, unit_type, player_id}` | 单位创建时 |
| `on_unit_destroyed` | `{unit_id, unit_name, player_id}` | 单位被摧毁时 |
| `on_unit_moved` | `{unit_id, from, to}` | 单位移动时 |
| `on_unit_attacked` | `{attacker, defender, damage}` | 单位攻击时 |

### 城市与建筑

| 钩子 | 数据 | 说明 |
|------|------|------|
| `on_city_founded` | `{city_id, city_name, player_id}` | 城市建立时 |
| `on_building_completed` | `{city_id, building_id}` | 建筑完工时 |

### 外交与事件

| 钩子 | 数据 | 说明 |
|------|------|------|
| `on_war_declared` | `{attacker, defender, reason}` | 宣战时 |
| `on_peace_signed` | `{party_a, party_b, terms}` | 和平时 |
| `on_trade_completed` | `{player_id, partner, amount}` | 交易完成时 |
| `on_diplomatic_event` | `{type, parties}` | 外交事件时 |

## 自定义钩子

模组可以触发自定义钩子，供其他模组监听：

```gdscript
# 模组 A：触发自定义事件
func _on_special_event():
    bridge.log_info("触发自定义钩子")
    bridge.trigger_hook("my_mod_special_event", {
        "value": 42,
        "message": "发生了特殊事件！"
    })

# 模组 B：监听自定义事件
func _ready():
    bridge.register_hook("my_mod_special_event", self, "_on_special_event")

func _on_special_event(data):
    bridge.log_info("接收到自定义事件: " + data.message)
```

## 钩子执行顺序

1. 基础游戏逻辑钩子
2. 核心模组钩子（按依赖顺序）
3. 依赖模组钩子
4. 普通模组钩子（按加载顺序）

同一优先级内，按注册顺序执行。

## 钩子数据修改

部分钩子支持通过修改数据字典来影响游戏行为：

```gdscript
func _on_damage_calculated(data):
    # 修改伤害计算
    data.damage = data.damage * 1.5
    bridge.log_info("伤害已增加 50%")
```

## 注意事项

- 钩子回调应尽量轻量，避免长时间阻塞
- 不要在同一帧内反复触发相同钩子
- 自定义钩子命名建议加模组前缀避免冲突
- 不再需要的钩子应及时取消注册
